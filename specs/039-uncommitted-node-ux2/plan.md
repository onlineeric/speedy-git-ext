# Implementation Plan: Uncommitted Node UX2 — Reuse File List in File Picker Dialog

**Branch**: `039-uncommitted-node-ux2` | **Date**: 2026-04-12 | **Spec**: [spec.md](spec.md)
**Input**: Feature specification from `/specs/039-uncommitted-node-ux2/spec.md`

## Summary

Replace the file picker dialog's plain file list (`FileGroup` component) with the same rich rendering used by the commit details panel — status badges, +N/−N line counts, list/tree view toggle — and add tri-state folder checkboxes in tree view. The view mode is shared bidirectionally with the commit details panel via the existing `fileViewMode` Zustand state.

## Technical Context

**Language/Version**: TypeScript 5.x (strict)
**Primary Dependencies**: React 18, Zustand, Radix UI (`@radix-ui/react-dialog`), Tailwind CSS
**Storage**: N/A — transient Zustand state + VS Code `globalState` for `fileViewMode` (existing persistence)
**Testing**: Vitest (unit tests for new utilities); manual smoke test via VS Code "Run Extension"
**Target Platform**: VS Code Extension (webview)
**Project Type**: VS Code extension (dual-process: esbuild backend + Vite/React frontend)
**Performance Goals**: Dialog open must feel at least as responsive as current; no perceived slowdown with several hundred uncommitted files
**Constraints**: No new persisted settings; no new backend RPC messages; no new packages
**Scale/Scope**: ~3-5 modified files, ~1-2 new shared component files; pure frontend change
